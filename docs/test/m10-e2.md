## 验证目标

1. 默认每小时自动检查可关闭；startup、robfig cron、wake 和 manual trigger 在同一 Coordinator 合并。
2. offline/429/坏 appcast 使用有界退避；配置/平台不可用 fail closed，重复触发不产生并发检查。
3. Wails 只暴露 finite DTO 与检查、下载、取消、跳过、稍后动作；64-bit 包大小使用十进制字符串跨端传输。
4. Settings 展示版本、摘要、包大小、签名和进度；下载需要显式确认且支持键盘焦点闭环。
5. 下载完成只显示 ready 状态；不调用最终 install reply，不越过 TOO-294 的 safe-drain 门禁。

## 失败与恢复

| 失败点 | 停止条件 | 恢复方式 |
| --- | --- | --- |
| Coordinator 并发或 race | 任何重复 check、锁竞争或 Close 未 drain | 保留 focused test，修复后重跑 `go test -race ./internal/updater ./internal/app` |
| binding drift | 生成 diff 超出 Go contract 或 typecheck 失败 | 使用固定 Wails 版本和 `-ts -i` 全量重生，不手改 generated files |
| UI 越过安装门禁 | 出现 install 命令/按钮或自动下载 | 立即移除并回到 TOO-292 frozen boundary |
| package gate 失败 | framework/version/rpath/signature 任一不符 | 停止 closeout，保留 ignored 产物诊断后完整重跑 |
