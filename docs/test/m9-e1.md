## 目标

- 验证冻结 Pencil/PNG 输入能重复生成完整 ICNS 与严格单色 Tray template。
- 验证 app 与 ZIP 实际包含固定资源名、正确 1x/2x 像素、alpha、thin arm64/minOS 15 和 ad-hoc 签名。
- 在本机启动 bundle，确认资源闭环不破坏应用启动；不验证后续 NSStatusItem 行为。

## 执行副作用

- DB / 用户数据：无；不读取或修改 Codex Home、应用数据库、Keychain 或凭据。
- 文件：`wails3 package` 会生成 ignored `bin/`、`frontend/node_modules/`、`frontend/dist/` 与 `.task/`；live smoke 将本轮 PID 写入 ignored `.task/m9-e1-app.pid`；测试临时 PNG 由 `testing.T.TempDir()` 自动清理。
- 服务：live smoke 会短暂启动 `Codex Pulse`，结束时终止并确认没有 9245 listener。
- 外部系统：无网络 API、Issue、release 或第三方系统触达；GitHub Actions 保持停用。
- 缓存：允许 Go/npm/Wails 构建缓存。

## 前置条件

- macOS 15+、Apple Silicon arm64。
- `wails3 version` 精确为 `v3.0.0-alpha2.117`。
- `go`、`node`、`npm`、`sips`、`iconutil`、`codesign`、`plutil`、`lipo`、`vtool`、`ditto` 可用。

## 测试变量与输入

- 源目录：仓库内 `docs/design/front/assets/icons`。
- package 输出：ignored `bin/Codex Pulse.app` 与 `bin/Codex Pulse.app.zip`。
- 版本输入：未发布基线 `0.0.0/0`；不创建 release archive。

## 步骤与预期结果

### 1. 源资产与 focused negative gate

```bash
go test ./build/darwin
go run ./build/darwin source docs/design/front/assets/icons
go run ./build/darwin render docs/design/front/assets/icons bin/.packaging/tray
go run ./build/darwin bundle bin/.packaging/tray
```

预期：测试覆盖彩色 template、尺寸漂移、全不透明输入等失败分支；六张源资产与两张派生资源读回通过。

### 2. Clean package 与 bundle 资源读回

```bash
wails3 task package:clean
wails3 package GOOS=darwin
wails3 task package:verify
```

预期：

- `Contents/Resources/icons.icns` 可反解十种标准 representation。
- `Contents/Resources/codex-pulse-tray-template.png` 为 19×19，`@2x` 为 38×38；两者都有透明/可见像素且所有可见像素 `R = G = B`。
- app/ZIP 继续满足 thin arm64、minOS 15、ad-hoc 签名与单顶层 archive 约束。

### 3. Live launch smoke

```bash
mkdir -p .task
rm -f .task/m9-e1-app.pid
before_pids=$(pgrep -x "Codex Pulse" || true)
open -n "bin/Codex Pulse.app"
app_pid=""
for pid in $(pgrep -x "Codex Pulse"); do
  if ! printf '%s\n' "$before_pids" | grep -qx "$pid"; then
    app_pid=$pid
    break
  fi
done
test -n "$app_pid"
printf '%s\n' "$app_pid" > .task/m9-e1-app.pid
test -z "$(lsof -nP -iTCP:9245 -sTCP:LISTEN || true)"
```

预期：进程可见、没有 Vite listener。此步骤只证明 bundle 可启动，不声称 Tray runtime 已由本卡实现。

### 4. 回归与 closeout gate

```bash
go test ./...
(cd frontend && npm test && npm run build)
make harness-verify
git diff --check
python3 .agents/skills/project-version-release/scripts/project_version_release.py check --repo "$PWD" --json
```

预期：全部通过；`CHANGELOG.md` 当前条目只位于 `Unreleased`，不产生版本段、tag 或 release。

## 清理

```bash
app_pid=$(cat .task/m9-e1-app.pid)
case "$app_pid" in (*[!0-9]*|'') echo "invalid app pid" >&2; exit 1;; esac
kill "$app_pid"
for _ in 1 2 3 4 5; do
  if ! kill -0 "$app_pid" 2>/dev/null; then break; fi
  sleep 1
done
if kill -0 "$app_pid" 2>/dev/null; then
  echo "本轮 Codex Pulse PID 未按期退出" >&2
  exit 1
fi
test -z "$(lsof -nP -iTCP:9245 -sTCP:LISTEN || true)"
rm -f .task/m9-e1-app.pid
wails3 task package:clean
```

- 只终止 `open -n` 后相对启动前新增的本轮 PID；既有同名进程保持不动。确认本轮 PID 和 9245 listener 均不存在。
- `frontend/dist/.gitkeep` 保留；其它 ignored 依赖/缓存可按需保留，不进入提交。
