## 验证目标

1. 固定 Sparkle 2.9.4 官方 release tools，并校验公开 archive、Mach-O 与 arm64 slice。
2. bundle 注入公开 feed URL/public key；private seed 只经 stdin 进入官方 `generate_appcast`。
3. 生成 ad-hoc macOS 15 arm64 app ZIP、同 basename release notes、单 item appcast 和审计 manifest。
4. verifier 不接触 private key，先验签再检查 ZIP 路径/类型/唯一 bundle，独立重算 ZIP/plist/appcast/notes/hash，并要求内嵌纯文本 notes 逐字一致。
5. release 内核独占锁覆盖共享 cache/bin/dist 且随崩溃自动释放；任一 pre-commit 失败不替换既有 `dist/update`，原子 swap 后的清理不反转成功结论，且不泄露 secret canary。

## 执行副作用

- `prepare_release_tools.sh` 写 ignored `.cache/sparkle/2.9.4/tools`；来源仍是已固定 SHA-256 的官方公开 archive。
- package 写 ignored `bin/`、`frontend/dist/`；local release 写 ignored `dist/update`。
- synthetic E2E 会重建上述 ignored 产物，并在失败注入阶段验证上一次 manifest 不变。
- 不读取真实 Keychain、secret manager、真实 feed/key 或用户数据。
