## 禁止字段与允许边界

| 边界 | 禁止 | 允许 |
| --- | --- | --- |
| parser / Store schema | 完整 prompt、response、tool output、raw JSON/content、Authorization/Cookie、access/refresh/auth token、raw error/message/stack | Session/Turn identity、时间、offset、usage、固定枚举、digest；索引所需 private source/project path metadata |
| public Wails DTO / health / event | 绝对路径、原始 identity、正文、凭据、raw error；event 中的业务事实 | 安全 attribution、整数统计、固定 class/code、content-free invalidation hint |
| frontend cache | Web Storage、IndexedDB 或 persisted query client 中的业务数据 | 仅进程内 TanStack Query cache；页面卸载/进程退出自然释放 |
| backup / crash / release / screenshot | synthetic body/token/path canary、credential envelope、真实用户内容 | 受控 metadata、签名/版本/架构/计数与固定 fault code |

字段审计按 token 化后的完整字段名判定，避免把安全 aggregate `promptTokens`、`responseBytes`、`contentLength`、`tokenCount` 或 `errorClass` 错判为正文/凭据字段。绝对路径采用分层 contract：private Store 可保存产品必需路径事实；任何返回前端、日志或提交证据的公共值都递归拒绝绝对路径。

## 失败与恢复

| finding | 行为 | 恢复 |
| --- | --- | --- |
| `forbidden_field` | 阻止审计和发布就绪 | 删除/重命名泄漏字段，重新生成 bindings，从 focused tests 重跑 |
| `sensitive_value` | 阻止审计，不回显命中值 | 清理本轮隔离 DB/backup/App，修复入库或 projection 边界后完整重跑 |
| `absolute_path` | 阻止公共 DTO/log/evidence | 将路径留在 private adapter，公共面改为安全 attribution 或固定 code |
| cache/artifact/startup log drift | 阻止 package 结论 | 删除持久 cache/raw log，重建 App/ZIP 并重新扫描全部 artifact |

runner 根使用 `0700`、固定 ownership marker 与唯一前缀；只有 marker、前缀和根身份都匹配才清理。package 使用独占 lock；若 `bin/codex-pulse`、App、ZIP、`.packaging` 或 lock 已存在则立即返回 `M11-PRIV-004`，不会覆盖或删除既有产物。任一检查失败时不会读取或修改真实用户数据，也不执行 token revoke、外部 API mutation 或发布 rollback。

## 重跑 Runbook

副作用：运行 synthetic tests；在系统临时目录创建私有 synthetic Home、Pure Go GORM SQLite 与 backup；从本机 module cache 构建临时 Wails CLI；本地生成 ad-hoc App/ZIP并扫描。退出时删除所有本轮临时根、App、ZIP和`.packaging`。若公开 framework cache已存在则只读复用；不会访问真实 Home/auth、用户数据库、Wham、Actions或发布系统。

```bash
make m11-privacy-audit
```

通过条件：最后一行 content-free JSON 必须同时满足 `result=passed`、`syntheticOnly=true`、全部 canary/path/log/cache/cleanup 布尔值为 `true`、schema/BLOB/generated/design-source/screenshot/workflow/package 计数非零；package gate 必须证明所有 symlink 受根约束，以及 arm64、minOS 15.0、ad-hoc codesign 与 ZIP 一致。任何缺项均 fail closed。

## 残余边界

- 本卡证明 synthetic canary 和当前 tracked/build artifact contract，不读取真实内容做 value matching；这是刻意的隐私边界，不是用 fixture 代替功能 E2E。TOO-298 已在用户明确只读授权下独立证明真实 Home 的 DB/DTO privacy，但其真实数据不进入本卡证据。
- 第三方 Sparkle framework regular files 接受同一 canary/envelope/path 扫描，symlink接受存在性与根内解析约束；不会声称对第三方二进制做源码级隐私证明。
- 当前产品没有诊断导出或云同步；若未来新增任何导出/上传/持久前端 cache，必须新增独立 Execution 与相应 privacy sink contract。
- Actions 依用户要求停用；不存在可下载的本轮 CI artifact。静态 workflow 合同通过不冒充 CI 运行通过。
