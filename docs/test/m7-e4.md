## 目标

验证 `/projects` 只消费 generated Wails DTO，支持本地日范围、归因置信度、排序与稳定 list pagination，并在独立详情区展示 provider 已对账的 aggregate、daily、Session contribution 和 Model contribution。任何 path、opaque identity、cursor、generation、conversation content 或底层 error cause 都不能进入 DOM。

## 执行副作用

- `npm ci/test/typecheck/build` 写入 npm cache、ignored `frontend/node_modules/` 与 `frontend/dist/`。
- `make verify` 写入 Go/build cache、ignored `.task/`、`bin/` 与生成校验产物。
- visual QA 会短暂构建隔离typed DTO版Wails app，并在 `.agents/runs/too-275-projects-qa/` 写 ignored screenshot/comparison。
- packaged QA 使用 isolated HOME/TMP 启动本机 app；结束后停止进程并清理已知隔离目录。
- normal-state 只使用不进入 production route 的 typed synthetic DTO fixture；不读取真实 Codex Home、SQLite、JSONL、Wham 或 credential。
- 不访问 GitHub Actions，不执行 release。
