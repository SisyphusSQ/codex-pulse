## 目标

- 用同一入口重复验证 idle/live/backfill/query/cleanup 资源路径，并记录 CPU、RSS、IO、DB/WAL、query latency 与 queue 事实。
- 注入权限、磁盘满/只读、SQLite lock、坏行、网络、休眠和进程中断，证明失败分类、保护与恢复不会污染真实数据。
- 成功标准：所有 phase 重复通过；数值门槛机械执行；环境、原始摘要、恢复结果和剩余风险可追溯。

## 执行副作用

- 文件：创建忽略的 `.agents/runs/<run-id>/` 原始痕迹；Go test 在临时目录创建并清理 synthetic 文件和数据库。
- DB：只打开 `testing.T.TempDir()` 下的 Pure-Go SQLite；DB/WAL/SHM、trigger/fault fixture 随测试结束删除，不打开默认应用数据库。
- 服务：网络测试只使用 Go fake transport/httptest contract；不启动常驻 server，不访问第三方服务。
- 系统：不执行真实 chmod、磁盘填充、断网、sleep 或进程 kill；异常退出只发生在测试自己创建的 child process。
- 缓存：允许更新 Go build/test cache；不触发 npm、GitHub Actions 或 release。

## 前置条件

1. Darwin arm64。
2. Go、Git、`sw_vers` 与 BSD `/usr/bin/time` 可用。
3. `MACOSX_DEPLOYMENT_TARGET` 未设置时由 harness 固定为 `15.0`；显式传入非 15.x 会 fail closed。
4. 当前工作树基于待验证分支；run id 只能包含 ASCII 字母、数字、`.`、`_`、`-`，且不得覆盖既有 run 目录。

## 平台说明与剩余风险

- 实际运行环境为 macOS `26.5.2` arm64；所有 phase 固定 `MACOSX_DEPLOYMENT_TARGET=15.0` 并通过，因此证明了 15.0 deployment target 的构建/链接与当前 arm64 runtime 行为。
- 本轮没有在实际 macOS 15 主机上运行；GitHub Actions 依用户要求保持 disabled，未查询、触发或等待。这个差异不得被描述成“macOS 15 runtime 已验证”。待存在授权的 macOS 15 主机时可复用同一 runbook补充 runtime 证据。
- evaluator 每次约 `2.51 MiB` transient allocation；本卡记录了重复 benchmark，但没有为 evaluator 单独设置 RSS/CPU threshold。若后续真实长期采样出现 GC 压力，再以 profiler-free allocation benchmark 单独调优，不在本卡猜测优化。

## 失败处理

| 失败点 | 停止条件 | 记录方式 | 恢复 / 重跑方式 |
| --- | --- | --- | --- |
| preflight | 非 Darwin arm64、非 15.x target、依赖缺失 | 不创建 run 目录，输出有限错误 | 修复环境后用新 run id 重跑 |
| 资源 phase | Go test/benchmark 失败或 timeout | manifest 写 FAIL，原始输出留本地 | 修复根因；从完整矩阵用新 run id 重跑 |
| 数值门槛 | benchmark unit 缺失或数值越界 | harness fail closed | 复核 workload/环境，优化后重复 5 次 |
| fault phase | 分类、保护或恢复断言失败 | 后续 phase 不标 PASS | 修复对应 adapter/contract，重跑完整矩阵 |
