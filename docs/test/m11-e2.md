## 真实数据发现与修复

1. 真实 Home 本轮约 6.55GB；原 256KiB chunk 和每条 quota observation 重建全窗口投影会造成大量重复事务。bootstrap 默认 chunk 提升到 1MiB；fast lane 内延迟 projection 并在发布 FirstScreenReady 前重建，backfill/reconcile 按 dirty window 持续刷新，final reconcile 后再重建并发布 FullHistoryReady。evidence 写入只更新 changed/stale 行，写后只读回目标 rows；公开读取仍从全部 immutable observations 重算并 fail closed。
2. 真实 rollout 中 turn ID 只在 session 内稳定。持久 Turn/Usage/SessionCurrent 改用 session-scoped canonical turn ID，parser checkpoint 继续保存原始 ID。
3. 真实 quota 事件可能按 source offset 后写入、但事件时间较早。source offset 仍严格单调；时间回退样本不再错误合并，而是保留独立 observation 交给 arbiter 判断。
4. 真实 session 可能同时存在多个 open turn。inactive turn 的 context/end 只更新自身事实，只有 active turn 才更新 SessionCurrent。
5. Project query 需要请求时区下的 active cost rollup。隔离验证器显式构建 UTC ledger 后，再用 UTC 时间范围验证公共 Project query；通用 bootstrap 不硬编码用户时区。
6. 大量 source 冻结记录使用 GORM 有界 `CreateInBatches`，规避 SQLite variable limit 并保持单事务回滚。
