## 目标

验证 `/overview` 仅消费 generated Wails query DTO，完整展示 quota、Token 趋势/构成、API 等价成本、每日明细、最近 Session/Project 与索引/健康摘要，并保持每个 query region 的 loading、known empty、partial/stale、fatal error/retry 相互隔离。

成功标准：

- 今天/7天/30天/自定义范围使用 IANA timezone 和本地日半开区间，最大 366 天；不固定 24 小时猜 DST。
- quota 保留 remaining、reset、来源、freshness 与 Reset credits；真实 0 不被当成 unknown。
- 趋势与每日明细直接使用服务端 `trend/totals`；microUSD 仅做显示换算，不从 daily rows 重算 aggregate 或 cost。
- 页面不展示 Session ID、dimension key、source key、路径、原始错误或其它本机敏感事实。
- ECharts 启用 Aria/decal、响应 resize、Reduce Motion 并在卸载时 dispose。
- source 与 implementation 在同一 1440×1024 viewport 对照后，无未关闭 P0/P1/P2 视觉 finding。

## 执行副作用

- `npm ci/test/typecheck/build` 会写入 npm cache、ignored `frontend/node_modules/` 与 `frontend/dist/`。
- 完整 `make verify` 会写入 Go/build cache、ignored `.task/`、`bin/` 与生成校验产物。
- visual QA 会短暂启动只监听本机 loopback 的 Vite server，并在 `.agents/runs/` 写入 ignored screenshot/comparison。
- realistic-data QA 使用隔离的临时 typed DTO cache；夹具不进入生产 route、产品运行路径或提交范围。
- 不访问真实 Codex Home、JSONL、SQLite、Wham、credential 或用户 Session；不触达外部 API、生产环境或 GitHub Actions。
- 原始命令输出与截图只保留在本地 ignored 结果面；提交版不包含完整本机 URL、绝对路径、凭据、内部 ID 或原始 error cause。
