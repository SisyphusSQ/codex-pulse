## 主路径步骤

1. 构造 cutoff 边界与 active/open/reference fixture，运行一批 cleanup 并核对各表行与 committed report。
2. 注入 batch failure/cancel，在 close/reopen 后无 cursor 重跑并核对最终收敛。
3. 产生 committed WAL 页，运行固定 PASSIVE checkpoint，核对 frame report 与并发 reader/writer 完成。
4. 用 fake clock/cron/sink 驱动 startup success、hourly due、failure backoff、overlap、panic 与 Close。
5. 运行 app composition、Pure-Go、race 与完整 integration funnel。

## 预期结果

- 所有删除只命中 frozen eligible rows；等于 cutoff、active/open/resumable/reference facts 保留。
- 任一事务错误不泄露未提交 count；重跑不需要恢复 cursor。
- PASSIVE checkpoint 不接受任意 mode/string，不开启第二 writer，不执行 truncate/vacuum。
- cleanup/checkpoint failure 只更新有限 projection 与退避，不终止 indexing/query/quota。

## 清理

- `testing.T.TempDir()` 由 Go test 自动清理。
- 测试 cron/context 必须 drain；不得残留 goroutine/lock。
- integration 产生的 `.task/`、`bin/`、`frontend/dist` ignored 产物验证后清理，保留 tracked `frontend/dist/.gitkeep`。
- Go/npm cache 可保留；不包含用户数据或凭据。

## 失败处理

| 失败点 | 停止条件 | 记录方式 | 恢复 / 重跑 |
| --- | --- | --- | --- |
| eligibility | 误删边界/open/reference row | 记录 synthetic case/count | 修复 GORM scope 后从新 temp DB 重跑 |
| batch | 超预算、部分 rollback count 外泄 | 记录 batch/report 摘要 | 修复事务与 report merge，重跑 fault matrix |
| checkpoint | 与 writer 并发、阻塞/任意 SQL | 记录 finite stage/frame | 修复 maintenance serialization，重跑 race |
| service | overlap、panic 后停摆、错误 due | 记录 finite projection | 修复 gate/backoff/Close，fake clock 重跑 |
| app | cleanup failure 影响主业务/关闭顺序 | composition test 失败 | 修复 lifecycle 后 focused race |
